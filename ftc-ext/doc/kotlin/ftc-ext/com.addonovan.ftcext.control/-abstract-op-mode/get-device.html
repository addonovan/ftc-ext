getDevice

inline fun <reified T : HardwareDevice> getDevice(name: String): T

A single method approach to getting a device, this requires a type parameter on the method, however, which means this cannot be correctly used if referenced in java.

Parameters

name - The name of the device.

Exceptions

IllegalArgumentException - If T had no DeviceMapping associated with it.

NullPointerException - If there was no entry for name with the type T.

IllegalAnnotationValueException - If the T class had a HardwareExtension annotation on it that had an invalid value (that is, one that doesnt have a value in the backing map of classes and DeviceMappings) assigned to the hardwareMapType parameter. This should never happen to an end-user, that would mean that the developer who wrote the extension did so wrongly.

IllegalClassSetupException - If T is a HardwareExtension that has been insufficiently set up, which could be for a multitude of reasons, check the error message if this occurs for more details.

Return
The value in the DeviceMapping in the hardware map for the given type, T, with the key name.
