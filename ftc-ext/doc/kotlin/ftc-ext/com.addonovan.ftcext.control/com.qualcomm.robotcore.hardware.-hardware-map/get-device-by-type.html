getDeviceByType

fun HardwareMap.getDeviceByType(type: Class<out HardwareDevice>, name: String): HardwareDevice

Gets the correct device from the Hardware device mappings in the HardwareMap by using the type of HardwareDevice to find the correct hardware device mapping and the name to get the correct value out of it.

Parameters

type - The type of hardware device to find.

name - The name of the hardware device to return.

Exceptions

IllegalArgumentException - If type had no DeviceMapping associated with it.

NullPointerException - If there was no entry for name with the type type.

IllegalAnnotationValueException - If the type class had a HardwareExtension annotation on it that had an invalid value (that is, one that doesnt have a value in the backing map of classes and DeviceMappings) assigned to the hardwareMapType parameter. This should never happen to an end-user, that would mean that the developer who wrote the extension did so wrongly.

IllegalClassSetupException - If type is a HardwareExtension that has been insufficiently set up, which could be for a multitude of reasons, check the error message if this occurs for more details.

Return
The hardware device with the given type and name.
